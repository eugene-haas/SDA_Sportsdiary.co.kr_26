# CodeRule of 위드라인

위드라인의 기본적인 코드규칙 입니다.

## 시작하기에 앞서...

본 규칙는 직원들의 협의를 통해 언제든 변경 가능합니다.<br/>
본 규칙을 통해 프로젝트의 **코드 품질**과 **유지보수 난이도**를 조절합니다.<br/><br/>

성능 향상 및 가독성 증가를 위해서 **본 규칙은 위배** 될 수 있습니다.<br/>
(단. 확고한 근거가 존재해야한다. - 주석으로 설명 필수)<br/><br/>

**기존 프로젝트의 코드 규칙을 따르는 것을 우선시**합니다.<br/>
(본 규칙은 앞으로의 프로젝트에 해당 / 2021-01-19 이후 프로젝트에 유효합니다.)<br/>

### 1-2. 함수명은 camelCase(lower) 사용
여러 단어가 합쳐 졌을때는 각 단어의 시작은 모두 대문자 (첫글자는 소문자)
~~~js
connectToTheServer();
sendToMetaData();
~~~

### 1-3. define 변수명은 모두 대문자로
~~~js
//대문자와 단어 사이에 '_' 를 붙인다.
const       G_START_YEAR         = 2020;        // 측정 검색 범위(2020 ~ currentYear)
const       G_PAGE_PER_CNT       = 30;          // 전역 리스트 개수 변수
const       G_PAGE_PER_CNT_MIN   = 7;           // 전역 리스트 개수 변수 - 화면 맞춤
const       G_MODAL_DURATION     = 300;         // 전역 모달 duration 0.3초
~~~
